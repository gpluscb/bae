instant-glicko-2

instant-glicko-2 is a Rust crate to calculate Glicko-2 ratings that update instantly after a game completes. Usually, you'd have to wait to the end of a rating period to have ratings update. I wrote about how to side-step that here, but that's mildly out of date now. The source code is on GitHub.

cache-compute

Another thing I stole from fasterthanlime? How wild is that. cache-compute is a Rust crate to cache results of async computations. I needed to cache some stuff for a personal project, but it turned out to be a harder problem than initially thought. If the cache is empty, you may want to start a computation. But you don't want to start one if someone else already did, you just want to be notified when that one is finished. Well luckily, at about the same time I // TODOd that, Amos published a very good article on the topic. And the implementation I created from that for my project grew decent enough to be factored out into a single-file crate, which I improved further and then open sourced on GitHub.

The coveted Idea List™

I'll organise some ideas from the list™ here. It's just some stuff that feels vaguely original, and that I might want to do at some point in the future.

The rules are simple: you can have a go at any and all of these if you want and however you want, but if you do, and it goes anywhere, you have to tell me about it. And in case you already know of some of these things existing, you also have to tell me immediately.

  • An autostereographic renderer. As in, something that can render 3D scenes to an autostereogram in real time. Bonus points for research into whether an autostereographic video game would make you barf immediately.
  • [whatever], colour picked from the [whatever] flag. You know these posts that are like "the trans flag, colour picked from this one Harmony Paruko render"? Automating that, but the other way around. I.e. a tool that can create an approximation of an image from any set of colours. I want to be able to put in an image of myself and get a reasonable approximation that uses only colours in the progress flag. Bonus points for utilizing dithering, that in particular seems really intimidating for arbitrary colours.
  • A declarative music making language. I don't really get DAWs. A lot of the time when I use them it feels a little like programming in Scratch. And don't get me wrong, I am 100% sure I am Dunning-Krugering the hell out of this. I have not spent the time to learn one of these, but I still really want to create a DAW-like thing that is less of a clicky GUI thingy and more of a programming language. Just that it compiles to an audio file instead of a binary. It sounds like it could be very cool, and really fun. Maybe a music making library could be a first step before actually trying to make it a whole language.
